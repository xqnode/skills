# 项目类型写法参考

用于在输出 `项目功能描述` 时，根据项目类型选择更贴切的角色命名、菜单名称和句式。  
本文件是写法参考，不是固定需求清单。若用户已明确指定角色、模块或菜单，以用户确认为准。  
若本文件中出现 `收藏`、`点赞`、`评价` 等客户端互动模块，默认视为候选项；只有在用户确认或场景高度匹配时才应写入最终需求文档，否则优先标记为“待确认”。

## 通用原则

- 优先按“角色 -> 固定账号功能 -> 菜单模块功能”的顺序输出
- 菜单名称要贴近系统场景，不要为了套模板而使用生硬名称
- CRUD 模块优先使用 `xxx管理：管理xxx信息`
- 非 CRUD 模块优先使用 `xxx模块：查看xxx、审核xxx、预约xxx、核销xxx、提交xxx`
- 数据统计优先使用 `数据统计：查看xxx、xxx折线图、xxx比例图`
- 若某角色没有注册、认证、审核等功能，不要硬写

## 健身房 / 场馆预约类

适用关键词：

- 健身房
- 健身工作室
- 瑜伽馆
- 游泳馆
- 羽毛球馆
- 预约到店

推荐角色：

- 管理员
- 教练 / 场馆工作人员
- 会员 / 用户

推荐模块写法：

### 管理员

- 登录
- 个人信息
- 修改密码
- 会员管理：管理会员信息
- 会员卡管理：管理会员卡信息
- 课程管理：管理课程信息
- 教练管理：管理教练信息
- 预约管理：查看预约信息
- 签到核销：查看签到记录、核销记录
- 基础信息管理：管理公告信息、场馆介绍等内容
- 数据统计：查看会员增长数据、预约折线图、到店比例图

### 教练 / 工作人员

- 登录
- 个人信息
- 修改密码
- 课程管理：查看本人课程安排
- 预约管理：查看预约名单
- 签到核销：核销到店信息
- 上课记录：查看授课记录、出勤记录

### 会员 / 用户

- 登录
- 注册
- 个人信息
- 修改密码
- 会员卡模块：查看会员卡信息、办理会员卡
- 课程模块：查看课程信息、教练信息
- 预约模块：预约课程、查看预约记录
- 签到模块：提交签到、查看签到记录
- 收藏模块（待确认）：收藏课程、查看收藏记录
- 点赞模块（待确认）：点赞课程内容、查看点赞记录
- 评价模块（待确认）：提交课程评价、提交教练评价、查看评价记录
- 公告模块：查看公告信息、场馆介绍

## 写作提醒

- 如果项目类型和这里的三类都不完全一致，优先复用句式，不要强行复用模块名
- 如果用户已经给出明确菜单名，直接沿用，不要再替换成“推荐模块”
- 如果用户只给出业务方向，没有给出模块名，可先按最贴近的项目类型组织草稿，再将未确认内容标记为“待确认”
- 若角色数量较少，例如只有 `管理员 + 用户`，管理员可写 6 到 8 个业务模块，用户可写 4 到 6 个业务模块，避免两边密度完全一样
- 若角色数量较多，例如 `管理员 + 教师 + 学生` 或 `管理员 + 商家 + 用户`，管理员保持最完整，业务角色写 3 到 5 个核心模块，普通用户写 4 到 6 个高频模块
- 非核心角色不要为了凑字数硬加模块，优先保留与主流程直接相关的菜单功能
- 角色命名优先使用行业角色名，如 `医生`、`卖家`、`买家`、`志愿者`、`居民`，不要全部退化成泛化的 `用户角色`、`业务角色`
- 宠物医院、宠物服务等细分场景下，优先使用 `宠物主人`、`宠物主` 等更自然的角色名，而不是泛化的 `用户`
- 若存在客户端角色，可将 `收藏`、`点赞`、`评价` 作为候选模块考虑；但要先判断该行业是否有自然的收藏对象、点赞对象和评价对象，且未确认时不要直接冻结进最终文档
