# 确认摘要到正式文档的映射规则

## project-spec.md

必须覆盖：
- 产品定位
- 目标用户与角色
- 项目目标
- 核心模块
- 核心业务流程
- 交付范围
- 明确不做项
- 答辩/演示重点

## db-design.md

只能从已确认的业务对象推导。

推荐顺序：
1. 角色相关表
2. 主业务表
3. 关系表或明细表
4. 已确认功能所需的支撑表

## api-contract.md

只能从已确认的业务流程推导。

优先覆盖：
- 用户侧主流程接口
- 管理侧审核与维护接口
- 已确认模块所需的列表、详情、新增、编辑、状态流转接口

## acceptance-criteria.md

必须满足：
- 可观察
- 可演示
- 与确认范围一致
- 与答辩重点一致

任何正式文档都不得加入未确认功能、猜测性表结构或猜测性接口。
